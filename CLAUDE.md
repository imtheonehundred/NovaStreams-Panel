## Architecture at a Glance

```
server.js (bootstrap — wires lib/boot/* modules)
  ├── lib/boot/db.js          MariaDB + Redis init, channel preload
  ├── lib/boot/routes.js      All route mounting, /health, /readyz
  ├── lib/boot/streaming.js   FFmpeg lifecycle, prebuffer, stability monitor
  ├── lib/boot/jobs.js        Cron jobs, Telegram bot, webhooks
  ├── middleware/             CSRF, Helmet+CSP, rate limiting, Joi validation, error handling
  ├── routes/                 56 Express routers (admin, reseller, client, xtream, stream, auth...)
  ├── services/               41 service modules (business logic)
  │   ├── serverSelectionService.js  Server load balancing / capacity
  │   ├── serverRuntimeService.js    Runtime session reconciliation
  │   ├── serverProxyService.js      URL resolution / proxy
  │   └── serverService.js           Compatibility facade (delegates to above)
  ├── lib/                    26 shared utilities (crypto, state, cache, panel-access, crons...)
  └── repositories/           21 raw-SQL modules (all parameterized, no injection risk)

Frontend:
  public/js/src/ (Vite source, ~50 lazy page chunks)
  → public/js/dist/ (build output — DO NOT commit, in .gitignore)

Streaming modes: node (FFmpeg → Node TS fan-out) OR nginx (HLS on disk)
Auth flow: /:accessCode gateway → express-session (Redis) → requireAuth / requireAdminAuth
Session: express-session + connect-redis, session.regenerate() on login (session fixation fixed)
CSP: Enabled — scriptSrc 'self', no unsafe-inline scripts
HSTS: Auto-enabled in production (NODE_ENV=production)
Stream auth: HMAC-signed URLs with TTL enforced in routes/stream.js
Audit: audit_log table + auditService wired to auth and admin CRUD
```

## Critical Rules for This Project

### Never Do
- Do NOT add `console.log` in production code — use `services/logger.js` (Winston)
- Do NOT write raw SQL outside of `repositories/` — all DB access through repository functions
- Do NOT hardcode secrets or config values — always use `process.env.*`
- Do NOT modify `public/js/dist/` manually — generated by `npm run build:frontend`
- Do NOT skip the CSRF token on mutating admin/auth endpoints
- Do NOT commit `.env` — it is in `.gitignore`
- Do NOT add inline `onclick`, `onchange`, `onsubmit` handlers to HTML files — CSP blocks them

### Always Do
- Use `lib/db.js` as the entry point for all data access (re-exports all repository functions)
- Validate request input with Joi via `middleware/validation.js` + `middleware/schemas/` before any business logic
- Use `lib/crypto.js` for all encryption/hashing — never use raw `crypto` module in routes
- Log errors via `services/logger.js`, not `console.error`
- Call `auditService.log(userId, action, resourceType, resourceId, meta, req)` on all admin state changes
- When adding a new admin page: route in `routes/admin.yourfeature.js` → mount in `routes/admin.js` → page in `public/js/src/pages/` → register in `lib/pageRegistry.js` → schema in `middleware/schemas/yourfeature.js`

## Environment Variables (Required)

The following env vars MUST be set or the server hard-fails at startup:

| Variable | Purpose |
|---|---|
| `SESSION_SECRET` | express-session signing key (64+ hex chars) |
| `LINE_PASSWORD_SECRET` | AES-256-GCM key for line password encryption |
| `STREAM_SECRET` | HMAC key for signed stream URLs |
| `DB_HOST`, `DB_USER`, `DB_PASSWORD`, `DB_NAME` | MariaDB connection |
| `REDIS_HOST`, `REDIS_PORT` | Redis connection (required — sessions live here) |
| `PUBLIC_STREAM_BASE_URL` | Base URL for stream/playlist URL generation |
| `STREAMING_MODE` | `node` or `nginx` |

**Never set `ALLOW_LOCAL_UNSIGNED_TS=1` in production** — bypasses stream token verification.
**Never use `change_me_*` values** from docker-compose defaults — server checks presence only, not strength.

## Database Rules

- InnoDB + utf8mb4 for all tables
- Timestamps: `DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP` (not INT UNSIGNED)
- Always add indexes for WHERE/JOIN/ORDER BY columns
- Schema changes: update `scripts/schema.sql` AND add migration to `scripts/migrations.js`
- Channel config stored as `json_data MEDIUMTEXT` with `version INT UNSIGNED` for optimistic locking

## Security Model Summary

| Concern | Implementation |
|---|---|
| Sessions | `express-session` + Redis, `session.regenerate()` on login |
| CSRF | Per-session secret, `X-CSRF-Token` on all mutating requests |
| CSP | `scriptSrc 'self'` — no inline scripts |
| HSTS | Auto-enabled in `NODE_ENV=production` |
| Rate limiting | 4 tiers (auth/admin/stream/apiKey), Redis-backed |
| Passwords | bcrypt cost 12 |
| Line passwords | bcrypt + AES-256-GCM |
| Stream URLs | HMAC-SHA256 signed with TTL |
| Audit log | `audit_log` table + `auditService` |
| Access code cache | Redis 60s TTL + per-request memoization |
| SQL injection | Parameterized queries throughout all 21 repositories |

## Scaling Constraint

The app is **single-process only** until P4.4 is completed. `lib/state.js` stores channel and FFmpeg process state in in-memory Maps that cannot be shared across PM2 cluster workers. Do not enable PM2 cluster mode until these Maps are migrated to Redis.
